<objective>
Integrate JWT authentication into the dashboard, add missing UI features (key expiry, per-key tool scoping, audit filters), and set up the build pipeline.

Purpose: The dashboard currently authenticates with raw admin secret sent as X-Admin-Secret header. Phase 5 built JWT auth on the backend (POST /admin/auth/login returns signed JWT, jwtAuthHook validates Bearer token). This plan wires the dashboard to use JWT, adds the missing DASH-05 (key scoping UI) and DASH-08 (audit filters) features, and establishes the build pipeline (DASH-09).

Output: Dashboard authenticates via JWT, all management features are functional, production build pipeline is ready.
</objective>

<interfaces>
From backend (src/server.ts):
```
POST /admin/auth/login
Body: { username: string, password: string }
Response 200: { token: string }
Response 401: { error: "Invalid credentials" }
```

Key creation endpoint (src/admin/router.ts):
```
POST /admin/tenants/:id/keys
Body: { label?: string, expiresAt?: string | null }
Response 201: { keyId: string, apiKey: string }
```

Per-key tool scoping (src/admin/router.ts):
```
PUT /admin/tenants/:id/keys/:keyId/tools
Body: { allowedTools: string[] | null }
Response 200: { updated: boolean }
```

List all tools:
```
GET /admin/tools
Response 200: string[]
```

Audit log query:
```
GET /admin/tenants/:id/audit-log?toolName=X&status=success|error&limit=N&offset=N
Response 200: { entries: AuditEntry[], totalCount: number }
```

Current ApiKey interface in dashboard:
```typescript
interface ApiKey {
  id: string;
  label: string | null;
  status: string;
  createdAt: string;
  revokedAt: string | null;
}
```
IMPORTANT: The backend getTenant() does NOT currently return `expiresAt` or `allowedTools` for API keys. The SELECT query in tenant-service.ts (line 175-181) omits these columns, and the ApiKeyRow interface (line 40-47) lacks them. The router response schema for GET /admin/tenants/:id also omits them. Task 2 must fix this on the backend first.

JWT auth hook (src/admin/auth-middleware.ts):
- Accepts `Authorization: Bearer <token>` header
- Falls back to `X-Admin-Secret` for backward compatibility
- Returns 401 with `{ error: "..." }` on invalid/expired token

  <name>Task 1: Rewrite dashboard auth to use JWT</name>
  <files>dashboard/src/api.ts, dashboard/src/App.tsx, dashboard/src/pages/LoginPage.tsx</files>
  <action>
  Three files need coordinated changes:

  **1. dashboard/src/api.ts** — Replace admin secret auth with JWT token auth:

  a) Replace `adminSecret` variable and its getter/setter/clearer with JWT token equivalents:
     - `let token = ''`
     - `setToken(t: string)` → stores in localStorage under key `'jwt_token'`
     - `getToken(): string` → reads from localStorage `'jwt_token'`
     - `clearToken()` → removes from localStorage and resets variable

  b) Add a `login(username: string, password: string): Promise<string>` function that:
     - Calls `fetch('/admin/auth/login', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ username, password }) })`
     - On success (200), extracts `{ token }` from response and returns the token string
     - On 401, throws `new Error('Invalid credentials')`
     - On other errors, throws with the error message from the response body

  c) Update the `api<T>()` function:
     - Change the header from `'X-Admin-Secret': secret` to `'Authorization': \`Bearer ${getToken()}\``
     - Keep the 401 handling (call `clearToken()` instead of `clearAdminSecret()`, then `window.location.reload()`)

  d) Keep all existing API function exports unchanged (listTenants, getTenant, etc.)

  e) Export the new functions: `login`, `setToken`, `getToken`, `clearToken`
  f) Remove old exports: `setAdminSecret`, `getAdminSecret`, `clearAdminSecret`

  **2. dashboard/src/pages/LoginPage.tsx** — Change to username + password form:

  a) Change the component props from `{ onLogin: (secret: string) => void }` to `{ onLogin: (username: string, password: string) => Promise<void> }`

  b) Add state for `username` (default: 'admin'), `password`, and `error` (string)

  c) Replace the single password input with two fields:
     - Username input (type="text", defaulting to 'admin')
     - Password input (type="password")

  d) On form submit:
     - Call `onLogin(username, password)` (which handles the API call in App.tsx)
     - On error, display the error message below the form

  e) Keep the existing Tailwind styling (centered card layout, blue submit button)

  **3. dashboard/src/App.tsx** — Wire JWT login flow:

  a) Import `login`, `setToken`, `getToken`, `clearToken` from `'./api'` (replace old imports)

  b) Change auth check from `!!getAdminSecret()` to `!!getToken()`

  c) Update the `LoginPage` `onLogin` callback:
     ```typescript
     async (username: string, password: string) => {
       const token = await login(username, password);
       setToken(token);
       setAuthenticated(true);
     }
     ```

  d) Update the `Layout` `onLogout` callback to call `clearToken()` instead of old clear function

  e) Update the storage event listener to check `getToken()` instead of old getter
  </action>
