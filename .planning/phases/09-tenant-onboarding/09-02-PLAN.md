**State management:**
- `step` state: 1 | 2 | 3 (current wizard step)
- Step 1 fields: `name` (string), `slug` (string), `plan` (string, default 'standard')
- Step 2 fields: `erpBaseUrl`, `erpClientId`, `erpAppSecret`, `erpUsername`, `erpPassword`, `erpTerminal` (all strings, default '')
- `connectionTested` (boolean, default false) — tracks whether credentials passed the test
- `testResult` ({ connected: boolean; message: string } | null) — last test result
- `testing` (boolean) — spinner state for Test Connection
- `submitting` (boolean) — spinner state for final Create
- `result` ({ tenantId: string; apiKey: string } | null) — creation result for step 3
- `error` (string) — error message display
- `copied` (boolean) — tracks whether API key was copied

**Stepper component (inline, not extracted):**
A horizontal stepper at the top showing 3 labeled steps. Render as an ordered list of step indicators:

```
const steps = [
  { num: 1, label: 'Tenant Info' },
  { num: 2, label: 'ERP Credentials' },
  { num: 3, label: 'API Key' },
];
```

Each step indicator shows: step number in a circle + label text below. Visual states:
- Completed step (num < current step): blue filled circle with white checkmark, blue label text
- Current step: blue filled circle with white number, blue bold label text
- Future step: gray border circle with gray number, gray label text

Render the stepper using a flex row with connecting lines (gray border-t) between circles.

**Step 1 — Tenant Info:**
- Fields: Name (text, required), Slug (text, required, lowercase/numbers/hyphens pattern with auto-sanitize onChange like current), Plan (select: standard/pro/enterprise)
- Slug auto-generation: when name changes AND slug has not been manually edited by the user, auto-generate slug from name by lowercasing and replacing non-alphanumeric with hyphens, trimming leading/trailing hyphens
- Buttons: "Cancel" (navigates to /tenants), "Next" (advances to step 2)
- Next button disabled unless both name and slug are non-empty
- Reuse exact same input/label/select classes as current implementation

**Step 2 — ERP Credentials:**
- Fields: all 6 ERP fields using the EXACT same field definitions as ErpTab (same labels, placeholders, types):
  - Base URL (type: url, placeholder: 'https://bigblue.posibolt.com')
  - Client ID (type: text, placeholder: 'OAuth client_id')
  - App Secret (type: password, placeholder: 'OAuth app_secret')
  - Username (type: text, placeholder: 'POSibolt username')
  - Password (type: password, placeholder: 'POSibolt password')
  - Terminal (type: text, placeholder: 'Terminal 1')
- Test Connection button: disabled if ANY of the 6 fields is empty OR if `testing` is true
- On Test Connection click: set `testing=true`, `testResult=null`, call `testErpCredentials({erpBaseUrl, erpClientId, erpAppSecret, erpUsername, erpPassword, erpTerminal})`, set `testResult` with response, set `testing=false`. On error, set `testResult = { connected: false, message: error.message }`
- Test result display:
  - Success: green bg `bg-green-50 border border-green-200`, green checkmark (unicode checkmark), text "Connection verified!"
  - Failure: red bg `bg-red-50 border border-red-200`, text from testResult.message (server includes hints from Plan 01)
- On any ERP field change: reset `connectionTested` to false, `testResult` to null (forces re-test)
- When `testResult?.connected === true`, set `connectionTested = true`
- Buttons: "Back" (goes to step 1, preserves all state), "Next" (advances to step 3)
- "Next" button ONLY enabled when `connectionTested === true`
- Test Connection button styled: `bg-blue-600 text-white` when enabled, shows "Testing..." with disabled state during test

**Step 3 — Create & API Key:**
- This step has TWO sub-states: before creation (submitting form) and after creation (showing result)

Before creation:
- Show a summary of what will be created: tenant name, slug, plan, and "ERP: Connected" with green checkmark
- "Create Tenant" button: on click, set `submitting=true`, call `createTenant({ name, slug, plan, erpBaseUrl, erpClientId, erpAppSecret, erpUsername, erpPassword, erpTerminal })`, on success set `result` with response
- On error: set `error` with message, set `submitting=false`
- Buttons: "Back" (goes to step 2, preserves state), "Create Tenant" (submits)

After creation (result is set):
- Green success banner: "Tenant created successfully!"
- Warning box (yellow bg `bg-yellow-50 border border-yellow-200`): "Save this API key now — it will not be shown again."
- API key display: monospace font in a bordered box, with a "Copy" button. On copy click, use `navigator.clipboard.writeText(result.apiKey)`, set `copied=true`, show "Copied!" for 2 seconds then revert to "Copy"
- Tenant ID shown as small text below
- Two buttons at bottom:
  - "View Tenant" — `bg-blue-600 text-white`, navigates to `/tenants/${result.tenantId}`
  - "Create Another" — `border border-gray-300 text-gray-700`, resets ALL state to initial values (step=1, all fields empty, connectionTested=false, result=null, etc.)

**Layout:**
- Container: `max-w-2xl` (wider than current max-w-lg to accommodate stepper)
- Title: "Create Tenant" as h1 with `text-2xl font-semibold text-gray-900 mb-6`
- Stepper below title, then step content below stepper
