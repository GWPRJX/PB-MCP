Architecture constraints:
- JWT_SECRET must come from env var, never hardcoded
- JWT signing: use Node.js built-in crypto (HMAC-SHA256) — no new dependencies
- JWT payload: { sub: 'admin', iat, exp }
- JWT expiry: configurable via JWT_EXPIRY_HOURS env var, default 8 hours
- Login credentials: ADMIN_USERNAME env var (default 'admin') + ADMIN_SECRET (existing env var, reused as password)
- After this plan: POST /admin/auth/login is UNPROTECTED; all other /admin/* routes accept EITHER X-Admin-Secret OR Authorization: Bearer JWT (backward compatible)
- Logging: stderr only (stdout corrupts MCP transport)
- Fastify hooks in a plugin scope apply to ALL routes in that scope (including those registered before the hook). Therefore the login endpoint MUST be registered OUTSIDE the adminRouter plugin scope.

  <action>
  1. **src/admin/auth-middleware.ts** — Create new file with JWT utilities using Node.js built-in `crypto` module (HMAC-SHA256). No external JWT library needed.

     Functions to export:

     `signJwt(payload: Record<string, unknown>): string`
     - Creates JWT header: `{"alg":"HS256","typ":"JWT"}`
     - Merges `iat` (current epoch seconds) and `exp` (iat + hours from `JWT_EXPIRY_HOURS` env var, default 8) into payload
     - Signs with `JWT_SECRET` env var using `crypto.createHmac('sha256', secret)`
     - Uses base64url encoding: `Buffer.from(json).toString('base64url')`
     - Returns `header.payload.signature` string
     - Throws if JWT_SECRET is not set

     `verifyJwt(token: string): Record<string, unknown> | null`
     - Splits token on '.', expects 3 parts
     - Recomputes signature from header.payload using JWT_SECRET
     - Uses `crypto.timingSafeEqual` for signature comparison (prevents timing attacks)
     - Checks `exp` against `Math.floor(Date.now() / 1000)`
     - Returns decoded payload on success, `null` on any failure

     `jwtAuthHook(request: FastifyRequest, reply: FastifyReply): Promise<void>`
     - Fastify onRequest hook
     - Checks `Authorization` header for `Bearer <token>` format — verifies via `verifyJwt()`
     - Falls back to `X-Admin-Secret` header — checks against `process.env.ADMIN_SECRET`
     - If neither valid: `reply.status(401).send({ error: 'Unauthorized', message: 'Valid JWT or admin secret required' })`
