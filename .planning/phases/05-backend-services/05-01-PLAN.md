<task type="auto">
  <name>Task 2: Create audit wrapper and wire into all tool handlers</name>
  <files>src/tools/errors.ts, src/tools/inventory.ts, src/tools/orders.ts, src/tools/crm.ts, src/tools/kb.ts</files>
  <action>
  1. **src/tools/errors.ts** — Add a `withAudit` higher-order function that wraps any MCP tool handler with audit recording. The wrapper must:
     - Accept `toolName: string` and the original handler function
     - Return a new handler with the same signature
     - Before calling the original handler, record `startTime = Date.now()`
     - Call the original handler, capturing the result
     - After the handler completes, extract tenantId and keyId from `tenantStorage.getStore()` (import from `../context.js`)
     - If context exists (tenantId + keyId available), call `recordToolCall(tenantId, keyId, toolName, params, status, errorMessage, durationMs)` where status is determined by checking `result.isError`
     - If context is missing (called outside tenant context), skip audit recording silently (this handles edge cases)
     - Duration is `Date.now() - startTime`
     - Return the original result unchanged
     - The wrapper must NOT throw — if recordToolCall fails, it already handles errors internally (fire-and-forget with stderr logging)
     - Import `recordToolCall` from `../admin/audit-service.js` and `tenantStorage` from `../context.js`

     Type signature:
     ```typescript
     type ToolHandler = (params: Record<string, unknown>) => Promise<{ isError?: boolean; content: { type: string; text: string }[] }>;
     export function withAudit(toolName: string, handler: ToolHandler): ToolHandler;
     ```

  2. **src/tools/inventory.ts** — For each of the 7 tool registrations, wrap the handler function with `withAudit('tool_name', handler)`. Import `withAudit` from `./errors.js`. The pattern for each tool changes from:
     ```typescript
     if (shouldRegister('list_products', filter)) server.tool(
       'list_products', description, schema,
       async (params) => { ... }
     );
     ```
     to:
     ```typescript
     if (shouldRegister('list_products', filter)) server.tool(
       'list_products', description, schema,
       withAudit('list_products', async (params) => { ... })
     );
     ```
     The handler body stays EXACTLY the same — only the wrapping changes.

  3. **src/tools/orders.ts** — Same pattern as inventory.ts. Wrap all 6 order tool handlers with `withAudit`.

  4. **src/tools/crm.ts** — Same pattern. Wrap all 5 CRM tool handlers with `withAudit`.

  5. **src/tools/kb.ts** — Same pattern. Wrap all 3 KB tool handlers with `withAudit`.

  Total: 21 tools wrapped. Each tool handler body is unchanged — only the `withAudit` wrapping is added.

  IMPORTANT: The `params` argument in the MCP SDK handler is destructured (e.g., `async ({ limit, offset })`). The `withAudit` wrapper receives the full params object before destructuring, so the audit log captures all params. This means the wrapper's handler type should accept the full params object. In practice, the MCP SDK passes the validated params object to the handler. The wrapper should capture `params` for audit logging before passing it to the inner handler.
