Create .github/workflows/ci.yml using the exact pattern from RESEARCH.md Pattern 8. The CI pipeline steps are:

1. actions/checkout@v4
2. actions/setup-node@v4 with node-version: '22', cache: 'npm'
3. `npm ci`
4. Install golang-migrate v4.19.1 (pinned — do NOT use @latest):
   ```bash
   curl -L https://github.com/golang-migrate/migrate/releases/download/v4.19.1/migrate.linux-amd64.tar.gz | tar xvz
   sudo mv migrate /usr/local/bin/migrate
   ```
5. Run migrations as the postgres superuser (DATABASE_URL in this CI step is the admin URL):
   `migrate -path db/migrations -database "$DATABASE_URL" up`
6. Create app_login role in the test DB (migrations create app_user NOLOGIN, but app_login needs LOGIN + a real password for the test DATABASE_URL):
   ```bash
   psql "$DATABASE_URL" -c "
     DO \$\$ BEGIN
       IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'app_user') THEN
         CREATE ROLE app_user NOLOGIN;
       END IF;
       IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'app_login') THEN
         CREATE ROLE app_login LOGIN PASSWORD 'ci_password';
       END IF;
     END \$\$;
     GRANT app_user TO app_login;
     GRANT CONNECT ON DATABASE pb_mcp_test TO app_user;
     GRANT SELECT, INSERT, UPDATE, DELETE ON ALL TABLES IN SCHEMA public TO app_user;
     GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA public TO app_user;
   "
   ```
   Note: This step may be redundant if migration 000001 runs as a superuser and creates the roles. Include it defensively.
7. Assert RLS coverage (INFRA-07 gate):
   ```bash
   bash scripts/assert-rls.sh
   ```
   This step must fail the build (exit 1) if any tenant-bearing table is missing RLS. The `set -euo pipefail` in assert-rls.sh ensures this propagates.
8. Run vitest:
   ```bash
   npm test
   ```
   Set environment for test step: DATABASE_URL (app_login) and DATABASE_MIGRATION_URL (postgres superuser). Both must be set so rls-isolation.test.ts can seed (superuser) and verify (app_user).

**Top-level CI configuration:**
- `name: CI`
- `on: [push, pull_request]`
- Single job named `test`, runs-on `ubuntu-latest`
- PostgreSQL service container: `postgres:17-alpine`, credentials `POSTGRES_USER: postgres`, `POSTGRES_PASSWORD: postgres`, `POSTGRES_DB: pb_mcp_test`
- Health check options using `--health-cmd pg_isready`, `--health-interval 10s`, `--health-timeout 5s`, `--health-retries 5`
- Port mapping: `5432:5432`

**Job-level env block:**
```yaml
env:
  DATABASE_URL: postgres://app_login:ci_password@localhost:5432/pb_mcp_test
  DATABASE_MIGRATION_URL: postgres://postgres:postgres@localhost:5432/pb_mcp_test
```

The full ci.yml should follow RESEARCH.md Pattern 8 exactly. Do not add extra steps not in the pattern. Do not add caching layers or matrix builds.
