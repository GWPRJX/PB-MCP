## Tool Access Control (TAC)

- [x] **TAC-01**: Admin can enable/disable specific MCP tools per tenant via REST API — *Phase 5 (05-01)*
- [x] **TAC-02**: Admin can restrict an API key to a subset of the tenant's enabled tools (per-key scoping) — *Phase 5 (05-01)*
- [x] **TAC-03**: Disabled/unscoped tools are not registered in the MCP server for that tenant's session — *Phase 5 (05-01)*
- [x] **TAC-04**: Every MCP tool call is recorded in an append-only audit log — *Phase 5 (05-01)*
- [x] **TAC-05**: Admin can query audit log filtered by tool name and status with pagination — *Phase 5 (05-01)*

## Auth & Security (AUTH)

- [x] **AUTH-01**: Dashboard login returns a JWT with expiry; dashboard stores JWT in localStorage — *Phase 5 (05-02)*
- [x] **AUTH-02**: API keys support optional expiry dates; expired keys rejected at auth time — *Phase 5 (05-02)*

- [x] **DASH-02**: Tenant list view showing name, slug, plan, status, key count — *Phase 6 (06-01)*

- [x] **DASH-06**: Tool Permissions tab — toggle tools on/off per tenant — *Phase 6 (06-01)*
- [x] **DASH-07**: ERP Config tab — update POSibolt credentials, test connection — *Phase 6 (06-01)*
- [x] **DASH-08**: Audit Log tab — paginated, filterable by tool name and status — *Phase 6 (06-01)*
