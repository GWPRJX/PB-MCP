# Customer Loyalty Administration & Redemption
This API manages all aspects of customer loyalty management including enrollment, points addition, rewards retrieval, and redemption processes.


**API Actions and Endpoints**

✔ Mandatory
-- Optional


| **Action**                                         | **URL**                                          | **Method** | **Parameters / Post Data**                                                                                    |
| -------------------------------------------------- | ------------------------------------------------ | ---------- | ------------------------------------------------------------------------------------------------------------- |
| Enroll customer to current default loyalty program | `/customerloyaltymaster/activateloyaltycustomer` | **POST**   | **Customer Model** (details of the customer enrolling to loyalty program)                                     |
| Add loyalty points for sales                       | `/customerloyaltymaster/addsalespoints`          | **POST**   | `loyaltyno*` (customer's loyalty number), `customerphone*` (customer phone number)  (**at least one required**). `amount` (Invoice amount)  ✔|
| Retrieve loyalty reward details                    | `/customerloyaltymaster/getloyaltydetails`       | **GET**    | `loyaltyno*`, `customerphone*`, `customerId*` (**either of these must be provided**)                          |
| Process loyalty payments & redeem points           | `/customerloyaltymaster/redeempoints`            | **POST**   | `loyaltyno*`, `customerphone*`, `customerId*` (**at least one required**), `paymentamt` (amount to redeem)  ✔   |
| Get all rewards                                    | `/customerloyaltymaster/rewards/getall`          | **GET**    | `limit` (max number of rewards to return)                                                                     |
| Get all loyalty types/programs                     | `/customerloyaltymaster/types/getall`            | **GET**    | `limit` (max number of types to return)                                                                       |
| Get all reward rules                               | `/customerloyaltymaster/rules/getall`            | **GET**    | `limit` (max number of rules to return)                                                                       |
| Get all loyalty accounts                           | `/customerloyaltymaster/accounts/getall`         | **GET**    | `updatedSince` (time in milliseconds, for fetching recently updated accounts)                                 |


**Details**
The ‘getall’ APIs shall be used to retrieve master data related to loyalty calculation. ‘Get all loyalty accounts’ API retrieves list of loyalty accounts/customer data, including points earned. ‘Get all loyalty types/programs’ is used to get list of loyalty programs active currently. ‘Get all rewards’ API is used to get list of rewards active currently. ‘Get all reward rules’ is used to retrieve rules applicable for a type/reward. 

Application shall use these data together to display correct set of rewards available for redemption for a customer/invoice.

To redeem points, applications need to create payments against invoice with tender type set as ‘loyalty’. Additionally, ‘loyaltyRewardId’ tag should be set based on selected reward to be redeemed and ‘productId’ tag if reward is of type product free/category free. Posibolt handles the redemption of appropriate rewards based on configuration.

Posibolt automatically handles adding loyalty points when sale is done, based on configuration, if customer is subscribed to an active loyalty program. The ‘Add loyalty points for sales’ need to be invoked only for invoices already synced to server, before loyalty program is made active.

Note: Loyalty point credit is added only for non-credit sales and when the shipment against sale is done.
