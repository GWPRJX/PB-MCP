# Product Search
* **URL** :`/productmaster/search`
* **Method**: GET
* **Description** : The API retrieves  product information, including pricing, stock availability, categorization, and warehouse-wise stock quantities.
* **Query Parameters**:
  1.`barcode`(**optional**) - (default value - empty string) - Return only products belonging to the given barcode.

  2.`searchText`(**optional**) - (default value - empty string)  -Return only products belonging to the given serach text.

  3.`priceListId`(**optional**) -Return only products belonging to the given pricelist Id.

  4.`productCategoryId`(**optional**) - Returns only the products belonging to the given productCategoryId.

  5.`excludeInTransit` **(optional):** Exclude stock quantities that are in transit. Default is false.

  6.`offset` **(optional):** Number of records to skip for pagination. Default is 0.

  7.`limit` **(optional):** Maximum number of records to return. Default is 0 (no limit).

  8.`updatedSince` **(optional):** Return products updated on or after the given timestamp (milliseconds since epoch).Default is 0.




NOTE:

* If **all fields** (barcode, searchText, priceListId, and productCategoryId) are empty or 0, the method will return `null` and the search will not be executed.
* If a value is entered in the **barcode** field and a UOM conversion exists for that product, the product list will be shown based on the converted UOM. If no UOM conversion is available, the product will be considered in its base UOM.
* If priceListId is passed, the specified price list will be checked, and the products whose prices are updated in that price list will be returned.
* If priceListId is not passed, the products will be returned based on the Sales Price List set in the terminal. If no price list is set in the terminal, the client’s default price list will be used.




**Sample Request:**

<http://127.0.0.1:8088/AdempiereService/PosiboltRest/productmaster/search?searchText=product>
