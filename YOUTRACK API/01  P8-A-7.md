# POSibolt REST-API
The POSibolt REST API is a composite API that gives programmatic access to master and transactional data, so that third-party applications can work as integrated software with POSibolt. With this composite API, you can create customer, order, invoice, shipment, and custom object records, etc., in a single call.

The POSibolt REST API can be used by any organization that has registered with that particular POSibolt instance.

Use the POSibolt REST API, a REST-based composite application programming interface, to:

- Create/Update Customer and Customer Groups Details  
- Create Order, Invoice, and Shipments  
- Create/Cancel Stock Transfer Requests  
- Create/Cancel Stock Receipts  
- Retrieve Stock Information

**Requirements and Limitations**

* Secure OAuth session ID required
* Max 2000 records per request
* Request/Response format: JSON
* All POST responses return JSON with status code & message
* Headers must include Content-Type: application/json
* Transactions are terminal-based
* Completed transaction records cannot be updated
* Each call must include an access token in the request header

**Authentication (OAuth 2.0)**

POSibolt uses OAuth2.

Steps:

* Register your application as a Connected App
* Get OAuth endpoint for your instance
* Authenticate using Username–Password flow
* Receive Access Token

**Request Example:**

`GET http://{server_url}/AdempiereService/oauth/token?username=myusername&password=mypassword&terminal=myterminal&grant_type=password`
`Authorization: Basic my_client_id:my_client_secret`


**Response Example:**

`{`
`  "access_token":"b17f91b6325a2c03c22dce6da6005974",`
`  "token_type":"bearer",`
`  "expires_in":43183`
`}`

**Base URL:**

`http://{servername}/AdempiereService/PosiboltRest`


**Headers:**

Accept: application/json
Authorization: Bearer <access_token>
