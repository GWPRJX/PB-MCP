# Peach Intergration Documents
# **Developer Integration Task – Peach Payments + Sunmi P2 SE**

### **Objective**

Integrate **Peach Payments’ Intent API (v2.0.3)** with the **Sunmi P2 SE** Android POS terminal to enable seamless, card-present transactions initiated directly from the POS application (Android or Web).

---

### **Scope of Work**

**1. Integration Sources**

* **Peach Payments Intent API v2.0.3 (PDF provided)**
* **Peach POS solution overview:** <https://www.peachpayments.com/solutions/point-of-sale>
* **API reference (Atlassian Wiki):** <https://peachpayments.atlassian.net/wiki/external/M2YxOTIyZGM2OTBhNDhiMWE0OGYyNjUzYWQ0MzA0YjQ>
* **Sunmi P2 SE device specs:** <https://www.sunmi.com/en/p2-se/>
* **Demonstration video:** <https://www.youtube.com/watch?v=TYej7zgP8lc>

---

### **2. Core Deliverables**

| Area | Task | Output |
| --- | --- | --- |
| **Web–to–App Integration (Webview)** | Implement the provided `web-wrapper` and define the web app URL in `WebActivity.kt`. | Configured `web-wrapper` app with embedded POS web app. |
| **Sale Transactions** | Implement `Sale.Builder` call with `SaleTransactionListener` callback. Handle `approved/declined` and error states. | Sale workflow in POS app and test approval/decline flows. |
| **Last Transaction Retrieval** | Implement `TransactionLookup.Builder` using provided examples. | Functionality to retrieve and display last transaction data. |
| **Transaction Search** | Support search by parameters (`RRN`, `Date`, `Card Last 4`, etc.). | Search interface in dev console or POS admin panel. |
| **Error Handling** | Map IntentError codes to POS error messages. | Central error-handler class with mappings. |
| **Device Setup** | Validate Peach Payment App installation and connectivity on **Sunmi P2 SE**. | Fully configured test terminal. |
| **Testing** | Simulate transactions using the sandbox environment. | Test results sheet with approvals, declines, and lookups verified. |

---

### **3. Key API Classes & Calls**

**Core Classes:**

* `Sale`, `Sale.Builder` – Initiate purchase transaction.
* `TransactionLookup.Builder` – For last transaction or transaction search.
* `PosTransactionSummary`, `PosTransactionResult`, `IntentError` – Handle responses.

**Supported Functions:**

* `startTransaction(amount)`
* `lastTransaction()`
* `searchTransaction(HashMap<PosTransactionLookupParameters, Any>)`

**Enum Parameters:**
`TRANSACTION_RESULT_COUNT`, `TRANSACTION_TENDER_TYPE`, `TRANSACTION_FROM_DATE`, `TRANSACTION_TO_DATE`, `TRANSACTION_AMOUNT`, etc.

### **5. Expected Outcomes**

* Working prototype on Sunmi P2 SE terminal executing end-to-end card transactions.

### **6. Acceptance Criteria**

\- Transaction initiation from POS triggers the Payment App via Intent.
- Terminal processes payment successfully and returns JSON response to POS.
- POS logs and displays the transaction result accurately.
- Lookup and search functions retrieve correct data.
- All critical error codes mapped and tested.
