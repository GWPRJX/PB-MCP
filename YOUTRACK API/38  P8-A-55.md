# Sales History
* **URL** : `/salesorder/saleshistory`
* **Method**: GET
* **Description** : The Api  allows syncing sales data from the retail POS and ERP systems to third party applications. The response contains detailed information about the sales, including customer details, invoice details, payment details, and products purchased.
* **Query Parameters:**

  1.`orgId` (**optional**): Query sales for a specific organization (store). Leave blank to query all organizations.

  2.`customerId` (**optional**): Query sales for a specific customer. Leave blank to query all customers

  3.`fromDate` (**optional**): Start date for querying sales data (format: YYYY-MM-DD).

  4.`toDate` (**optional**): End date for querying sales data (format: YYYY-MM-DD).

  5.`includeCRO` (**optional**): Fetches sales data including returned orders. The default value is **true**. Set to ‘false’ to exclude returned orders.

  6.`Offset`(**optional**): The number of records to skip. Use this for pagination, with the default being **0**.

  7.`limit`(**optional**): The maximum number of records to return. The default is **100**, and the value cannot exceed this limit.




NOTE:

* If **all fields** (orgId, customerId, fromDate, and toDate) are empty or 0, the method will return error message("Either orgId, customerId, or date range (fromDate and toDate) are required.") and the search operation will not be performed.

  

**Sample Request:**

<http://127.0.0.1:8088/AdempiereService/PosiboltRest/salesorder/saleshistory?fromDate=2026-03-03&customerId=1018641>
